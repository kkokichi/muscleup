# 01. 要件定義

## 1. プロダクト概要

**MuscleUp** — 「ジムに行くのが楽しくなる」筋トレ記録アプリ。

筋トレ × SNS × AIコーチ × ゲーミフィケーションを組み合わせ、
**継続率の最大化**を最優先のプロダクトゴールとする。

### コアバリュー

| 体験 | 提供手段 |
|---|---|
| 記録する | 3タップ以内で完了する超高速ログ入力 |
| 成長を実感する | 重量推移グラフ・1RM推定・自己ベスト・継続日数 |
| 仲間と励まし合う | SNS機能（フェーズ2以降） |
| AIに応援される | AIマスコットによる文脈対応メッセージ |

## 2. ターゲットユーザー

- **メイン**: 筋トレ初心者〜中級者（ジム歴0〜3年）。記録が続かず挫折した経験がある層
- **サブ**: 上級者。詳細なログと分析を求める層

## 3. 機能要件

### 3.1 MVP（フェーズ1）

| ID | 機能 | 概要 | 優先度 |
|---|---|---|---|
| F-01 | トレーニング記録 | 日付・種目・重量・回数・セット・RPE・メモ・所要時間を記録。履歴閲覧 | P0 |
| F-02 | 重量推移 | 種目ごとの重量推移 / 推定1RM / 自己ベスト / 月間成長率のグラフ表示 | P0 |
| F-03 | 種目辞典 | フォーム動画(YouTube)・やり方・効く部位・注意点・初心者ポイント・よくある失敗 | P1 |
| F-04 | 部位マップ | 人体イラストをタップ→部位別の種目一覧 | P1 |
| F-05 | AIマスコット | 応援・継続促進・記録称賛・自己ベスト祝福。文脈に応じたランダムメッセージ | P1 |
| F-06 | ホーム画面 | 今日の予定 / 最近の記録 / 週間回数 / 継続日数 / AIコメント / レベル / PB更新数 | P0 |

### 3.2 フェーズ2以降（設計のみ考慮）

- **SNS**: チェックイン、投稿、コメント、いいね、フォロー、写真/動画投稿、DM、コミュニティ、ランキング
- **AI**: フォーム解析、疲労分析、メニュー提案、停滞期分析、重量予測、食事アドバイス、AIチャットコーチ
- **ゲーミフィケーション**: 経験値、レベル、称号、バッジ、クエスト、連続ログイン/筋トレ日数
- **その他**: Apple Health / Watch / Google Fit 連携、ジム検索、食事・体重管理、有料プラン

## 4. 非機能要件

| 項目 | 要件 |
|---|---|
| 保守性 | Clean Architecture。UI / Feature / Service / Repository / Types の層分離 |
| 拡張性 | Repository パターンでデータソースを抽象化（Local ⇄ Firestore を差し替え可能） |
| パフォーマンス | 初期表示 2秒以内。入力操作は即時反映（Optimistic UI） |
| オフライン | MVPはローカル保存で完全オフライン動作。Firestore接続時は自動切替 |
| デザイン | Apple HIG 準拠。黒基調＋ライムグリーンアクセント。ダークモード標準 |
| 対応環境 | モバイルファーストのレスポンシブWeb（PWA化を視野） |

## 5. UX原則

1. **3タップ以内で記録完了** — 前回の重量・回数を自動プリセット
2. **入力は最小限** — ステッパー（＋/−）でワンタップ調整、キーボード入力は任意
3. **即時フィードバック** — 保存時にマスコットが反応、PB更新は即時祝福
4. **毎日開きたくなる** — ホームに継続日数・週間サマリー・今日のひとことを常設

## 6. 成功指標（KPI）

- 週次アクティブ継続率（4週後リテンション）
- 1セッションあたりの記録完了率
- 記録開始→完了までの平均タップ数 / 秒数

## 7. スコープ外（MVP）

- ユーザー登録・ログインUI（匿名/ローカル利用で開始。Firebase Auth は基盤のみ用意）
- SNS・AI解析・課金
