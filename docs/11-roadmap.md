# 11. 将来拡張計画

## フェーズ構成

### フェーズ1: MVP（実装済み）
記録・進捗・辞典・マップ・マスコット・ホーム。ローカル保存で完全動作。

### フェーズ1.7: 部位マップ詳細化＋コミュニティ（実装済み）
- 部位マップを18の筋肉（大胸筋上/中/下部など）に細分化し、種目を31に拡充
- ジムチェックイン（Google Maps＋位置＋コメント）
- 種目アドバイスの共有（投稿・いいね）
- 共有系は Repository 抽象化により Firestore（共有）/ ローカル（単一端末）を自動切替
  → 設定手順は [13-backend-setup.md](13-backend-setup.md)

### フェーズ1.8: ジム中UX改善（実装済み）
- ワークアウトテンプレート（前回メニュー・保存済みメニューから開始）
- セット完了時に自動開始するレストタイマー（60/90/120秒）
- 履歴詳細からの記録編集・削除、編集/削除後の自己ベスト再計算
- PWA対応（manifest・ホーム画面追加用アイコン・standalone表示）

### フェーズ1.5: アカウント & 同期
- Firebase Auth（匿名→Google/Apple連携のアップグレードパス）
- Firestore 同期（ローカルデータのマイグレーションウィザード）
- オフラインキャッシュの高度化（Service Worker戦略、データ同期キュー）

### フェーズ2: ゲーミフィケーション & SNS基盤
- XP/レベルの本格運用（Cloud Functions でサーバー権威化）
- バッジ・称号・クエスト（`achievements` マスタ駆動。条件は conditionType で宣言的に定義済み）
- 連続ログイン/筋トレ日数の通知リマインド
- ジムチェックイン、投稿（記録の自動サマリーカード共有）、いいね・コメント・フォロー
- 通知センター、週間ランキング

### フェーズ3: AI
- AIチャットコーチ（Claude API。記録データをコンテキストに応援・助言）
- 今日のメニュー提案（疲労分析: 部位別の直近ボリュームから回復状態を推定）
- 停滞期検知と打開提案（推移データのトレンド分析）
- 重量予測（1RM推移の回帰）
- フォーム動画解析（動画アップロード → 姿勢推定）
- 食事アドバイス

### フェーズ4: エコシステム
- Apple Health / Apple Watch / Google Fit 連携
- ジム検索（Gyms マスタ + 地図）
- 食事・体重・サウナ記録
- 大会モード、パーソナルトレーナーマッチング
- 有料プラン（AI機能・詳細分析を Pro 化）、広告

## アーキテクチャ上の布石（実装済みの拡張ポイント）

| 拡張 | 布石 |
|---|---|
| Firestore移行 | Repository Factory。環境変数のみで切替 |
| ゲーミフィケーション | levelService の XP計算を差し替え可能に分離。achievements スキーマ定義済み |
| SNS | WorkoutLog が Post から参照可能なID設計。ER図に全テーブル定義済み |
| AIコーチ | mascotService の Message供給元を静的配列→API に差し替え可能 |
| 多言語化 | 種目マスタに nameJa/nameEn を併記済み |
